しかし我々のSolidityコントラクトでは`owner`はSolidityの`address`であることを予期している。どうやってアプリユーザーのアドレスを知ることができるのだろうか？

## Metamaskでユーザーのアカウントを取得する

MetaMaskの拡張機能内で、ユーザーは複数アカウントの管理が可能だ。

次のように、どのアカウントが現在アクティブなのかをインジェクトされた`web3`の変数上で確認することができる:

```
var userAccount = web3.eth.accounts[0]
```

ユーザーはMetamaskのアクティブアカウントをいつでも切り替え可能だから、我々のアプリはこの変数が変更されているかをモニタリングして、それに応じてUIをアップデートする必要がある。例えばユーザーのホームページがゾンビ軍団を表示する場合に、ユーザーがMetamaskのアカウントを変更したら、選択された新アカウントのゾンビ軍団を表示するようにページを更新したい。

これは次のように`setInterval`のループで可能だ:

```
var accountInterval = setInterval(function() {
  // アカウントが変更されているかチェック
  if (web3.eth.accounts[0] !== userAccount) {
    userAccount = web3.eth.accounts[0];
    // 新アカウントでUIをアップデートするように関数コール
    updateInterface();
  }
}, 100);
```

これは、100ミリ秒ごとに`userAccount`が`web3.eth.accounts[0]`とイコールのままかをチェックしている（例:ユーザーがまだそのアカウントをアクティブにしているか）。もし違った場合は、`userAccount`を現在のアクティブアカウントに割り当て直し、表示をアップデートする関数を呼び出す。

我々のアプリは、ページが最初にロードされるときにユーザーのゾンビ軍団を表示し、またMetaMaskのアクティブなアカウントをモニタリングして変更があれば表示を更新するようにしたい。

1. `userAccount`という名の`var`を宣言せよ。ただし何も割り当てないように。

2. `startApp()`の最後に、上の`accountInterval`のテンプレートコードをコピー＆ペーストすること。

3. `updateInterface();`の行を`getZombiesByOwner`へのコールで置き換えて、`userAccount`を渡せ。

4. `getZombiesByOwner`の後に`then`ステートメントを続け、その結果を`displayZombies`という名前の関数に渡せ(シンタックスはこうだ: `.then(displayZombies);`)。
