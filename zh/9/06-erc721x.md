要了解向用户转移代币时在`awardToken`中会发生什么，我们来看一下`ERC721XToken.sol`中普通转移函数发生的情况；

### 转移同质化代币

`ERC721XToken.sol` 中的同质化代币（FT）转移函数的代码如下：

```
function _transferFrom(address _from, address _to, uint256 _tokenId, uint256 _amount)
    internal
    isOperatorOrOwner(_from)
{
    require(tokenType[_tokenId] == FT);
    require(_amount <= balanceOf(_from, _tokenId), "Quantity greater than from balance");
    require(_to != address(0), "Invalid to address");

    _updateTokenBalance(_from, _tokenId, _amount, ObjectLib.Operations.SUB);
    _updateTokenBalance(_to, _tokenId, _amount, ObjectLib.Operations.ADD);
    emit TransferWithQuantity(_from, _to, _tokenId, _amount);
}
```

如你所见，这个函数的主要是将发送方的代币余额减少`_amount`，并将接收方的代币余额增加相同的`_amount`。

但是因为我们还有标准版卡牌，所以`awardToken`函数逻辑需要略有不同。

记住，我们还有无限量供应的标准版卡牌。我们将其存储为`0`供应量，并且不想从`0`中减去`_amount`。

因此，我们的`awardToken`逻辑将会检查这是张限量版卡牌还是供应量为`0`的标准版卡牌 —— 并且只减去限量版卡牌的代币余额。

2. 如果`if`语句为真，我们首先要确保游戏服务器的这张卡牌有足够的剩余量来发送给用户，否则就给出错误提示。我们可以用以下代码来检查：`require(_amount <= balanceOf(msg.sender, _tokenId), "Quantity greater than remaining cards");`
