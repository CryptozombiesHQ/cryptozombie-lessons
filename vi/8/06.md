Now we're going to implement the third game mode, *Zombie Pile*. Again, we've forked the previous contract into a new file called `zombiepile.sol` as a starting point, but you can still reference `singleton.sol` in its own tab.

As a refresher on the rules of this game mode:

> **Zombie Pile**: At the start of each match, both players' cards are shuffled together, and they each randomly get a mix of their own deck and their opponents deck.

The way we're going to implement this logic is a bit different than actually shuffling the decks together and then dealing all the cards out as a secondary step.

Instead, we're going to do it in one step by iterate through both players decks card by card (as we did in the previous implementations). And for each card, we're going to flip a coin (generate a random number that's either 0 or 1). If the number is `0`, we'll add the card to Player 1's deck. If it's `1`, we'll add it to Player 2's deck. 

This will have the same effect as shuffling the decks together would, but is easier to conceptualize in code.

Since this logic is a bit different from the previous implementation, we'll need to instantiate `newCards` for both Player 1 and Player 2 *before* the `for` loop. That way we can add each card to one of the two decks within each loop iteration, instead of doing them one at a time for each player.
