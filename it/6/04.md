Il nostro contratto è pronto! Ora possiamo usare Web3.js per parlarci.

Web3.js ha due metodi che possiamo usare per chiamare le funzioni nel nostro contratto: `call` e `send`.

### Call

`call` è usato per le funzioni `view` e `pure`. Funziona solo sul nodo locale e non creerà una transazione sulla blockchain.

> **Revisione:** le funzioni `view` e `pure` sono di sola lettura e non cambiano stato sulla blockchain. Inoltre non costano alcun gas e all'utente non verrà richiesto di firmare una transazione con MetaMask.

Usando Web3.js dovresti usare `call` per una funzione chiamata `myMethod` con il parametro `123` come segue:

```
myContract.methods.myMethod(123).call()
```

### Send

`send` creerà una transazione e cambierà i dati sulla blockchain. Dovrai usare `send` per tutte le funzioni che non sono `view` o `pure`.

> **Nota:** il `send` (invio) di una transazione richiederà all'utente di pagare il gas e farà apparire Metamask che chiederà di firmare la transazione stessa. Quando usiamo Metamask come nostro provider web3, tutto questo sarà automatico e non abbiamo bisogno di fare nulla di speciale nel nostro codice. Abbastanza bello!

Usando Web3.js dovresti inviare una transazione che richiama la funzione chiamata `myMethod` con il parametro `123` come segue:

```
myContract.methods.myMethod(123).send()
```

La sintassi è quasi identica a `call()`.

## Ottenere i Dati degli zombi

Ora diamo un'occhiata ad un esempio reale dell'utilizzo di `call` per accedere ai dati del nostro contratto.

Ricordiamoci che abbiamo reso la nostra serie di zombi `public`:

```
Zombie[] public zombies;
```

In Solidity quando dichiari una variabile `public`, si creerà automaticamente una funzione pubblica "getter" con lo stesso nome. Quindi se volessi cercare lo zombi con id `15`, lo richiameresti come se fosse una funzione: `zombies(15)`.

Ecco come scriveremo una funzione JavaScript nel nostro front-end per richiede uno zombi tramite il suo ID, farà una query (richiesta) sul nostro contratto per quello zombi e ne restituirà il risultato:

> Nota: tutti gli esempi di codice che utilizziamo in questa lezione utilizzano la **versione 1.0** di Web3.js, che utilizza promises anziché callbacks. Molti altri tutorial che vedrai online utilizzano una versione precedente di Web3.js. La sintassi è cambiata molto con la versione 1.0, quindi se stai copiando il codice da altri tutorial, assicurati che stiano usando la tua stessa versione!

```
function getZombieDetails(id) {
  return cryptoZombies.methods.zombies(id).call()
}

// Chiama la funzione e fai qualcosa con il risultato:
getZombieDetails(15)
.then(function(result) {
  console.log("Zombie 15: " + JSON.stringify(result));
});
```

Vediamo cosa sta succedendo qui.

`cryptoZombies.methods.zombies(id).call()` comunicherà con il nodo del provider Web3 e gli dirà di restituire lo zombi con indice `id` da `Zombie[] public zombies` del nostro contratto.

Si noti che questo è **asincrono**, come una chiamata API a un server esterno. Quindi Web3 restituisce una promise (promessa) qui. (Se non hai familiarità con le promise di JavaScript... È tempo di fare alcuni esercizi prima di continuare!)

Una volta che la promise si risolve (il che significa che abbiamo ricevuto una risposta dal provider web3), il nostro codice di esempio continua con l'istruzione `then` che registra `result` sulla console.

`result` sarà un oggetto JavaScript che assomiglia a questo:

```
{
  "name": "H4XF13LD MORRIS'S COOLER OLDER BROTHER",
  "dna": "1337133713371337",
  "level": "9999",
  "readyTime": "1522498671",
  "winCount": "999999999",
  "lossCount": "0" // Obviously.
}
```

Potremmo quindi avere una logica di front-end per analizzare questo oggetto e visualizzarlo in modo significativo sul front-end.
