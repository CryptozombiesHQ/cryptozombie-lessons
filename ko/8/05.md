이제 지난 챕터에서 건너뛴 단계의 중간 로직을 채워볼 것이네:

> 2. `newCards`에 추가해둔 카드가 이미 덱에 있는 카드인지 하나하나 확인하게. 이미 있는 카드라면, `cardAlreadyInDeck`을 `true`로 설정하게

카드가 이미 덱에 있는지 확인하는 가장 쉬운 방법은 두 카드의 `mouldName`을 비교하는 것이네 — 여러 장의 같은 카드는 같은 `mouldName`을 가진다네.

> 참고: mouldName은 카드의 이름이라네. 예를 들어, "Zhampion" 또는 "Burrrn"과 같이 말이지. 우리는 이것을 displayName과 구분하기 위해 `mouldName`이라고 부른다네. 이렇게 하면 개발자들은 그들이 단순히 카드의 겉으로 보이는 이름을 바꾸는 것이라고 생각하면서 이 속성을 변경하지 않는다네 🙃

하지만, 크립토좀비 코스 1을 상기해보면 (혹은 자네가 이미 능숙한 솔리디티 개발자라면) — 솔리디티에는 문자열을 비교할 수 있는 기본적인 함수가 없다네.
 그래서 우리는 두 문자열이 같은지 비교하기 위해서는 그들의 `keccak256` 해시를 비교해야 한다네:
 ```
if (keccak256(abi.encodePacked(newCards[k].mouldName)) == keccak256(abi.encodePacked(gameState.playerStates[i].cardsInDeck[j].mouldName))) {
  // 이 카드들은 같은 `mouldName`을 가지는군!
}
```

이것을 조금씩 구현해 보도록 하지.

## 직접 해보기

1. `uint k`를 0으로 설정하고, `k < cardCount`까지 반복하는 `for` 반복문을 선언하게.

2. `for` 반복문 내에서는, 위에서 언급한 것과 같이 `newCards[k].mouldName`와 `gameState.playerStates[i].cardsInDeck[j].mouldName`의 `keccak256` 해시가 같은지 비교하는 `if` 구문을 만들게. 

3. 이 구문이 참이라면, `cardAlreadyInDeck = true;`를 설정하게.

다 되었군! 이 반복문이 끝난 후, 해당 카드가 `newCards`에서 발견되었다면, `cardAlreadyInDeck`은 `true`, 발견되지 않았다면 `false`가 될 것이네. 그리고 우리가 지난 챕터에서 구현한 로직은 `cardAlreadyInDeck`이 `false`일 경우에만 해당 카드를 덱에 추가할 것이네.

이렇게 하면 Singleton의 로직이 완성된 걸세!
