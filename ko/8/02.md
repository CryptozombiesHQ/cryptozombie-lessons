이제 우리 컨트랙트를 위한 뼈대를 갖추었으니, 규칙들을 바탕으로 플레이어들의 카드를 필터링하는 커스텀 로직을 짜야 하네.

우리는 `Munchkin`의 규칙에 따라, 비용이 2 goo 이상인 카드를 필터링할 것이네.

가장 먼저 해야 할 일은 각 플레이어의 카드를 순회하는 걸세. 지난 레슨에서 다음과 같은 중첩 `for`문을 사용하여 이 작업을 수행했던 것을 기억해보게나.


```
for (uint i = 0; i < gameState.playerStates.length; i++) {
    CardInstance[] memory newCards = new CardInstance[](gameState.playerStates[i].cardsInDeck.length);
    uint cardCount = 0;

    for (uint j = 0; j < gameState.playerStates[i].cardsInDeck.length; j++) {
        if (isLegalCard(gameState.playerStates[i].cardsInDeck[j])) {
            newCards[cardCount] = gameState.playerStates[i].cardsInDeck[j];
            cardCount++;
        }
    }

    changes.changePlayerCardsInDeck(Player(i), newCards, cardCount);
}
```

이것은 자네의 커스텀 게임 모드에서 흔히 보게 될 로직의 구조라네. 각 플레이어의 덱이 자네의 커스텀 게임 규칙을 따르고 있는지 검증하기 위한 것이지. 자네는 이 로직을 복사하여 자네의 수많은 커스텀 게임 모드들에 붙여넣을 수 있다네.

## 직접 해보기

1. 규칙을 따르는 카드를 걸러내기 위한 필터를 추가할 수 있도록, 자네의 컨트랙트에 위에 주어진 중첩 `for`문을 복사해서 붙여넣게.

2. `isLegalCard`라는 이름의 함수를 정의하게. 이 함수는 `card`라는 이름을 가진 `CardInstance`를 인자로 받고, `bool` 타입을 반환하네. 이 함수는 `internal` 및 `view` 속성을 가져야 하네.

  지금은 함수의 내용을 비워두게.
