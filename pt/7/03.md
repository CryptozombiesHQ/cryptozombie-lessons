Neste capítulo, vamos desconstruir aquele desagradável array de bytes, `serializedGameState`, em algo que possamos facilmente ler e interagir.

Uma das estruturas auxiliares que temos acesso em `ZBGameMode.sol` é chamada de` GameState` - e é uma estrutura de dados que contém todos os dados dentro de `serializedGameState`.

Vamos dar uma olhada no que há dentro.

## `GameState`

`GameState` é uma` struct` que contém todo o estado da partida atual - e todas as variáveis ​​que podemos querer alterar. Se parece com isso:

Mais tarde, veremos como interagir com esta estrutura `GameState`. Mas primeiro vamos inicializá-lo com os dados de `serializedGameState`.

## Inicializando um `GameState`

A primeira coisa que você quer fazer é declarar um novo `GameState` em memória:

```
GameState memory gameState;
```

Então, queremos usar a função `init` para ler todos os dados de` serializedGameState` e usá-lo para preencher nosso `gameState` com dados:

```
gameState.init(serializedGameState);
```

> Nota: A lógica por trás de `init` está em` ZBSerializer.sol`, e é um pouco complicada. Mas isso ocorre nos bastidores e você não precisa se preocupar em entendê-lo para criar um modo de jogo personalizado - a menos que você seja um super nerd assim😉
>
> A parte importante é que a lógica principal do jogo Zombie Battleground está usando exatamente o mesmo formato de codificação e decodificação do `ZBSerializer`, então as duas partes do jogo são capazes de se comunicar e passar um estado de jogo inteiro para frente e para trás.

Agora os dados são preenchidos em `gameState` e poderemos acessar o estado atual do jogo usando as subpropriedades de` gameState`.

Por exemplo, se você quiser ver qual é o nome da primeira carta do baralho do Jogador 1, você pode fazer isso com:

```
gameState.playerStates[0].cardsInDeck[0].mouldName
```

Ou você pode verificar a defesa Overlord atual do Jogador 2 com:

```
gameState.playerStates[1].defense
```

Veremos isso em capítulos futuros, conforme precisamos usá-los.
